# 注册/登录 500 错误排查指南

## 一、可能原因

1. **数据库表结构与当前模型不一致**（之前用旧模型建过表）
2. **MySQL 未启动或连接失败**
3. **数据库账号/密码错误**

---

## 二、解决步骤

### 步骤 1：查看具体报错（推荐先做）

在**运行 `python app.py` 的那个终端**里，每次 500 错误会打印完整错误信息。请查看是否有类似：

- `OperationalError: (pymysql.err.OperationalError) ...` → 数据库连接问题
- `ProgrammingError: ... column 'xxx' doesn't exist` → 表结构与模型不一致

### 步骤 2：重置数据库表（表结构不符时用）

若怀疑是表结构问题，可先重置表：

```bash
cd "/Users/houyuexian/Downloads/小狗钱钱app开发/code/backend"
export MYSQL_PASSWORD=你的密码
python reset_db.py
```

会先删除所有表，再按当前模型重新建表。然后再测试注册/登录。

### 步骤 3：开启调试模式查看错误

```bash
cd "/Users/houyuexian/Downloads/小狗钱钱app开发/code/backend"
export MYSQL_PASSWORD=你的密码
export FLASK_DEBUG=1
python app.py
```

再执行测试脚本或手动调用注册接口，500 时会在返回的 `msg` 里看到具体错误信息。

### 步骤 4：确认 MySQL 状态

```bash
brew services list | grep mysql
# 若未启动：
brew services start mysql
```

---

## 三、推荐执行顺序

```bash
cd "/Users/houyuexian/Downloads/小狗钱钱app开发/code/backend"

# 1. 设置数据库密码
export MYSQL_PASSWORD=123456

# 3. 启动后端

```

然后在另一个终端执行：

