# 接口测试说明

## 一、已实现的 MySQL 存储和日记功能

### 1. 数据库存储

- **users 表**：存用户（手机号、加密密码）
- **diaries 表**：存成功日记，每条记录包含：
  - `content`：日记内容
  - `date`：日期（如 2026-03-10）
  - `category`：分类（work/health/relationship/growth/daily）
  - `mood_icon`：情绪图标
  - 以及 `user_id`（归属哪个用户）

### 2. 日记接口（需先登录）

| 接口 | 作用 |
|------|------|
| `GET /api/v1/diary/entries` | 查看日记列表（支持分页、按日期/分类筛选） |
| `POST /api/v1/diary/entries` | 新建一条日记 |
| `PUT /api/v1/diary/entries/:id` | 修改某条日记 |
| `DELETE /api/v1/diary/entries/:id` | 删除某条日记 |

## 二、接口通不通的完整流程

整体顺序：

```
1. 启动后端 (python app.py)
2. 健康检查 (确认服务在跑)
3. 发送验证码
4. 注册 (或直接登录)
5. 登录拿到 token
6. 用 token 调用「创建日记」
7. 用 token 调用「查看日记」
```

## 四、手动用 curl 测试（理解每一步）

### 步骤 1：健康检查

```bash
curl http://localhost:5001/health
```

期望：`{"code":200,"msg":"success","data":{"status":"healthy"}}`

### 步骤 2：发送验证码

```bash
curl -X POST http://localhost:5001/api/v1/auth/sms/send \
  -H "Content-Type: application/json" \
  -d '{"phone":"13800138000"}'
```

期望：`{"code":200,"msg":"验证码发送成功","data":null}`

### 步骤 3：注册

开发环境验证码固定为 `123456`。

```bash
curl -X POST http://localhost:5001/api/v1/auth/register \
  -H "Content-Type: application/json" \
  -d '{"phone":"13800138000","sms_code":"123456","password":"test123456"}'
```

期望：返回 `code:200`，`data` 里有 `token` 和 `refresh_token`。

若该手机号已注册，改做登录：

```bash
curl -X POST http://localhost:5001/api/v1/auth/login \
  -H "Content-Type: application/json" \
  -d '{"phone":"13800138000","password":"test123456"}'
```

### 步骤 4：创建日记（需要 token）

把下面命令里的 `你的token` 换成步骤 3 返回的 `data.token`：

```bash
curl -X POST http://localhost:5001/api/v1/diary/entries \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer 你的token" \
  -d '{"content":"今天完成了一件小事，很开心","date":"2026-03-10","category":"daily","mood_icon":"Sun"}'
```

期望：`{"code":200,"msg":"success","data":{"diary_id":1}}`

### 步骤 5：查看日记列表

```bash
curl "http://localhost:5001/api/v1/diary/entries?page=1&page_size=10" \
  -H "Authorization: Bearer 你的token"
```

期望：`data` 里有 `total` 和 `list`，`list` 中有刚创建的日记。

## 五、核心概念说明

1. **Token**：登录成功后返回的字符串，后续所有需要「已登录」的接口，都要在请求头带上 `Authorization: Bearer <token>`。
2. **统一返回格式**：`{ "code": 200, "msg": "success", "data": {...} }`，`code` 为 200 表示成功。
3. **验证码**：开发环境下发送验证码后，验证码固定是 `123456`。
