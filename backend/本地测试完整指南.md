## 一、先搞懂几个概念（用大白话说）

### 1. MySQL 是什么？

**MySQL** 就是一个「存数据的仓库」。  
你的 App 要存用户、日记、愿景等内容，不能只放内存里（关掉就没了），需要放到数据库里，持久保存。

- 可以把它想象成一个超级表格软件。
- 你的后端（Flask）负责「往里写、往外读」这些数据。

### 2. CRUD 是什么？

**CRUD** 就是四种基本操作：

| 英文 | 中文 | 意思 |
|------|------|------|
| **C**reate | 创建 | 往数据库里「新增」一条记录 |
| **R**ead   | 读取 | 从数据库里「查」数据 |
| **U**pdate | 更新 | 修改已有的记录 |
| **D**elete | 删除 | 删除一条记录 |

你朋友说「写个 CRUD 测数据库」，意思是：先写一段小程序，能对数据库做增删改查，如果都能成功，就说明数据库连接没问题，后面可以放心写业务逻辑。

### 3. 整体流程是啥？

```
① 在电脑上安装 MySQL（把仓库建起来）
      ↓
② 创建「数据库」和「用户名/密码」（给仓库配钥匙）
      ↓
③ 改 Flask 配置，让它能连上这个数据库
      ↓
④ 写一个简单的 CRUD 测试（增删改查）
      ↓
⑤ 测试通过 → 再写登录、日记等业务
```

## 二、第一步：安装 MySQL

### 在 Mac 上安装（用 Homebrew，最简单）

#### 2.1 检查是否装了 Homebrew

打开终端，输入：

```bash
brew --version
```

- 如果显示版本号（比如 `Homebrew 4.x.x`）→ 说明已装，跳到 2.2。
- 如果提示 `command not found` → 先装 Homebrew，在终端执行：

```bash
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"
```

按提示操作，装完后再执行 `brew --version` 确认。

#### 2.2 安装 MySQL

在终端执行：

```bash
brew install mysql
```

等待安装完成（可能需要几分钟）。

#### 2.3 启动 MySQL 服务

```bash
brew services start mysql
```

看到 `Successfully started mysql` 就说明启动成功。

#### 2.4 设置 root 密码（第一次安装时）

```bash
mysql_secure_installation
```

- 问 `Validate Password Component` 时，可以先选 `n`（不启用密码强度校验）。
- 问 `Set root password` 时，选 `Y`，输入一个你能记住的密码（比如 `123456`，本地测试够用）。
- 后面几个问题可以一路回车选默认。

## 三、第二步：创建数据库和用户

### 3.1 登录 MySQL

```bash
mysql -u root -p
```

输入你刚设的 root 密码，回车。  
如果成功，会看到类似 `mysql>` 的提示符。

### 3.2 创建数据库和用户

在 `mysql>` 提示符下，**逐行复制粘贴**执行（注意分号不能漏）：

```sql
-- 创建数据库
CREATE DATABASE qianqian_diary CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;

-- 创建用户（用户名 qianqian，密码 123456，可改成你喜欢的）
CREATE USER 'qianqian'@'localhost' IDENTIFIED BY '123456';

-- 给这个用户授权，可以操作 qianqian_diary 这个库
GRANT ALL PRIVILEGES ON qianqian_diary.* TO 'qianqian'@'localhost';
FLUSH PRIVILEGES;

-- 退出
EXIT;
```

> 解释：  
> - `qianqian_diary`：数据库名，专门给你这个项目用。  
> - `qianqian` / `123456`：后端连接数据库用的账号密码，可以改成别的，但要和后端配置一致。

## 四、第三步：修改后端配置，让它连上数据库

### 4.1 用环境变量配置（推荐）

在终端执行（**把 `123456` 换成你给 qianqian 设的密码**）：

```bash
cd "/Users/houyuexian/Downloads/小狗钱钱app开发/code/backend"

export MYSQL_HOST=localhost
export MYSQL_PORT=3306
export MYSQL_USER=qianqian
export MYSQL_PASSWORD=123456
export MYSQL_DATABASE=qianqian_diary
```

说明：
- `localhost`：数据库就在本机。
- `3306`：MySQL 默认端口。
- 用户名、密码、库名要和你在 MySQL 里创建的一致。

### 4.2 或者直接改 config.py

打开 `backend/config.py`，找到这几行，改成你实际的值：

```python
MYSQL_HOST = "localhost"
MYSQL_PORT = 3306
MYSQL_USER = "qianqian"
MYSQL_PASSWORD = "123456"   # 改成你的密码
MYSQL_DATABASE = "qianqian_diary"
```

## 六、第五步：写一个简单的 CRUD 测试

### 6.1 什么是「测试」？

就是加一个专门的接口，让你能用「创建、读取、更新、删除」来验证数据库是否正常工作。

### 6.2 测试接口说明

我们会在后端加一个 `/api/test/crud` 路径，用来做简单测试：

- **POST**：创建一条测试数据
- **GET**：读取刚才创建的数据
- **PUT**：更新这条数据
- **DELETE**：删除这条数据

只要这四个都能成功，就说明 CRUD 和数据库都没问题。

---

## 七、第六步：启动后端并验证

### 7.1 启动 Flask

```bash
cd "/Users/houyuexian/Downloads/小狗钱钱app开发/code/backend"
python app.py
```

看到 `Running on http://0.0.0.0:5001` 就说明启动成功。

### 7.2 用浏览器或 curl 测试

**健康检查：**

- 浏览器打开：`http://localhost:5001/health`
- 应看到：`{"status":"healthy"}`

**CRUD 测试（需要按顺序）：**

1. **创建**：在终端新开一个窗口执行  
   ```bash
   curl -X POST http://localhost:5001/api/test/crud -H "Content-Type: application/json" -d '{"name":"测试用户","email":"test@example.com"}'
   ```
   应返回新建的记录（含 id）。

2. **读取**：  
   ```bash
   curl http://localhost:5001/api/test/crud
   ```
   应看到刚才那条记录。

3. **更新**：把上面返回的 `id` 换成你的，比如 id=1  
   ```bash
   curl -X PUT http://localhost:5001/api/test/crud/1 -H "Content-Type: application/json" -d '{"name":"新名字"}'
   ```

4. **删除**：  
   ```bash
   curl -X DELETE http://localhost:5001/api/test/crud/1
   ```

如果这几步都成功，就说明：**数据库连上了，CRUD 也能正常工作**。  
接下来可以放心写登录、日记等业务逻辑。

## 八、常见问题

### Q1：`mysql: command not found`

- 说明 MySQL 的命令没加到环境变量。  
- 可以试试：`/opt/homebrew/bin/mysql -u root -p`（Apple Silicon  Mac）  
- 或：`/usr/local/bin/mysql -u root -p`（Intel Mac）

### Q2：`Can't connect to local MySQL server`

- MySQL 没启动，执行：`brew services start mysql`

### Q3：`Access denied for user 'qianqian'`

- 检查用户名、密码是否和 `config.py` 或环境变量里的一致。
- 检查是否执行了 `GRANT` 和 `FLUSH PRIVILEGES`。

### Q4：`ModuleNotFoundError: No module named 'xxx'`

- 依赖没装全，在 `backend` 目录执行：`pip install -r requirements.txt`

## 九、总结：你现在要按顺序做的事

1. 安装 MySQL（`brew install mysql`）
2. 启动 MySQL（`brew services start mysql`）
3. 设置 root 密码（`mysql_secure_installation`）
4. 登录 MySQL，创建数据库和用户（复制上面的 SQL 执行）
5. 修改 `config.py` 或设置环境变量
6. 执行 `python init_db.py` 建表
7. 执行 `python app.py` 启动后端
8. 用 curl 或浏览器测试 CRUD 接口

做完这些，你的本地环境就完全打通了，可以开始写业务逻辑了。
