# Qianqian Diary 后端部署指南

本文档说明如何在腾讯云轻量服务器上部署 Flask 后端项目。

---

## 一、环境准备

### 1. 系统要求

- 操作系统：Ubuntu 20.04+ 或 CentOS 7+
- Python 3.9+
- MySQL 5.7+ 或 8.0+

### 2. 安装 Python 与 pip

**Ubuntu/Debian：**

```bash
sudo apt update
sudo apt install python3 python3-pip python3-venv -y
```

**CentOS：**

```bash
sudo yum install python3 python3-pip -y
```

### 3. 安装 MySQL

**Ubuntu：**

```bash
sudo apt install mysql-server -y
sudo mysql_secure_installation
```

**CentOS：**

```bash
sudo yum install mysql-server -y
sudo systemctl start mysqld
sudo grep 'temporary password' /var/log/mysqld.log  # 获取初始密码
```

创建数据库和用户：

```sql
mysql -u root -p

CREATE DATABASE qianqian_diary CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
CREATE USER 'qianqian'@'localhost' IDENTIFIED BY '你的强密码';
GRANT ALL PRIVILEGES ON qianqian_diary.* TO 'qianqian'@'localhost';
FLUSH PRIVILEGES;
EXIT;
```

## 二、修改配置

### 1. 数据库连接

编辑 `config.py` 或通过环境变量配置：

| 环境变量 | 说明 | 示例 |
|----------|------|------|
| MYSQL_HOST | 数据库地址 | `localhost` 或 `127.0.0.1` |
| MYSQL_PORT | 端口 | `3306` |
| MYSQL_USER | 用户名 | `qianqian` |
| MYSQL_PASSWORD | 密码 | `你的强密码` |
| MYSQL_DATABASE | 数据库名 | `qianqian_diary` |

**方式一：直接改 config.py**

```python
MYSQL_HOST = "localhost"
MYSQL_USER = "qianqian"
MYSQL_PASSWORD = "你的密码"
MYSQL_DATABASE = "qianqian_diary"
```

**方式二：使用环境变量**

```bash
export MYSQL_HOST=localhost
export MYSQL_USER=qianqian
export MYSQL_PASSWORD=你的密码
export MYSQL_DATABASE=qianqian_diary
```

### 2. 密钥（生产环境必改）

```bash
export SECRET_KEY="随机生成的长字符串"
export JWT_SECRET_KEY="可同 SECRET_KEY 或单独设置"
```

## 三、安装依赖

### 1. 创建虚拟环境（推荐）

```bash
cd backend
python3 -m venv venv
source venv/bin/activate   # Linux/Mac
# 或 Windows: venv\Scripts\activate
```

### 2. 安装依赖

```bash
pip install -r requirements.txt
```

依赖包括：

- `flask`：Web 框架
- `flask-sqlalchemy`：ORM
- `pymysql`：MySQL 驱动
- `PyJWT`：JWT 生成
- `flask-cors`：跨域支持

## 五、运行项目

### 1. 开发模式（直接运行）

```bash
python app.py
```

应用会在 `0.0.0.0:5000` 启动，支持公网访问。

### 2. 生产模式（Gunicorn 推荐）

安装 Gunicorn：

```bash
pip install gunicorn
```

启动：

```bash
gunicorn -w 4 -b 0.0.0.0:5000 "app:app"
```

- `-w 4`：4 个工作进程
- `-b 0.0.0.0:5000`：监听 0.0.0.0，端口 5000

### 3. 使用 systemd 守护进程（可选）

创建 `/etc/systemd/system/qianqian-api.service`：

```ini
[Unit]
Description=Qianqian Diary API
After=network.target mysql.service

[Service]
Type=simple
User=www-data
WorkingDirectory=/path/to/backend
Environment="PATH=/path/to/backend/venv/bin"
Environment="MYSQL_HOST=localhost"
Environment="MYSQL_USER=qianqian"
Environment="MYSQL_PASSWORD=你的密码"
Environment="MYSQL_DATABASE=qianqian_diary"
Environment="SECRET_KEY=你的密钥"
ExecStart=/path/to/backend/venv/bin/gunicorn -w 4 -b 0.0.0.0:5000 "app:app"
Restart=always

[Install]
WantedBy=multi-user.target
```

启用并启动：

```bash
sudo systemctl daemon-reload
sudo systemctl enable qianqian-api
sudo systemctl start qianqian-api
```

## 六、验证

### 1. 健康检查

```bash
curl http://localhost:5000/health
# 预期: {"status":"healthy"}
```

### 2. 注册与登录

```bash
# 1. 发送验证码
curl -X POST http://localhost:5000/auth/sms/send \
  -H "Content-Type: application/json" \
  -d '{"phone":"13800138000"}'

# 2. 注册（开发环境验证码为 123456）
curl -X POST http://localhost:5000/auth/register \
  -H "Content-Type: application/json" \
  -d '{"phone":"13800138000","code":"123456","password":"test123456"}'

# 3. 密码登录
curl -X POST http://localhost:5000/auth/login \
  -H "Content-Type: application/json" \
  -d '{"phone":"13800138000","password":"test123456"}'
```

## 七、HTTPS 与 Nginx（生产环境）

生产环境建议用 Nginx 做反向代理并配置 HTTPS：

```nginx
server {
    listen 80;
    server_name your-domain.com;

    location / {
        proxy_pass http://127.0.0.1:5000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}
```

使用 Let's Encrypt 配置 HTTPS：

```bash
sudo apt install certbot python3-certbot-nginx -y
sudo certbot --nginx -d your-domain.com
```

## 九、常见问题

1. **pymysql 连接失败**  
   - 检查 MySQL 是否启动：`sudo systemctl status mysql`  
   - 检查账号密码、数据库名是否正确  
   - 检查防火墙是否放行 3306（仅本机连接时一般不需要）

2. **CORS 错误**  
   - 当前配置已允许所有域名，生产环境建议在 `config.py` 中限制 `CORS_ORIGINS` 为实际前端域名

3. **验证码**  
   - 当前为开发模拟，验证码固定为 `123456`  
   - 生产需接入阿里云、腾讯云等短信服务
